# 🧾 **DevBind — Local Dev SSL Reverse Proxy (Native Rust / Dioxus App)**

A desktop application for Linux that manages local DNS overrides, issues trusted SSL certificates, and runs a reverse proxy for multiple local development services — all in a native UI.

##  **Project Vision**

DevBind provides developers with an easy way to map local hostnames to services with HTTPS support — without DNS servers, manual certs, nginx configs, or browser warnings. It integrates everything into one native app.

---

#  **Core Features**

##  1. Local Domain Management

You define human-friendly domains for local services.

### Requirements

* Write only specific entries in `/etc/hosts` safely
* Mark DevBind blocks so it never overwrites unrelated entries
* Detect and prevent conflicting entries

### UI

* List of hostname → port mappings
* Add / edit / delete entries
* Real-time validation

### Example

| Domain         | Port | Status |
| -------------- | ---- | ------ |
| sub1.dev.local | 8085 | ✔      |
| sub2.dev.local | 3210 | ✔      |
| sub3.dev.local | 3333 | ✔      |

---

##  2. SSL Certificate Management

DevBind will provide trusted SSL locally for any defined domain.

### Supported Modes

1. **mkcert-based TLS**

   * Use external mkcert to generate & trust certs
   * Trusted by browsers automatically
2. **Built-in cert gen (rcgen)**

   * Self-signed certs
   * Not trusted by browsers by default
   * Useful for environments without mkcert

### Features

* Wildcard certificates support (`*.dev.local`)
* Per-hostname cert generation
* Certificate renewal UI

### UI

* Show cert status: Valid / expires in X days
* Auto-renew buttons

---

##  3. Reverse Proxy Engine (Rust)

DevBind incorporates a reverse proxy that listens for HTTPS and forwards internally.

### Key Goals

* Support HTTP/HTTPS
* Map each domain to a backend port
* Efficient and async

### Tech

* `hyper`
* `rustls`
* TLS terminated here

---

##  4. Secure Port & System Integration

By default, Linux needs root to bind port 443 and edit hosts.

### Supported Options

* **Direct root** via helper binary
* **Port redirect via iptables** (recommended)

  * Bind high port (8443) and redirect 443 to 8443
* **Privilege escalation**

  * `pkexec` prompt only when needed
  * Lowest privilege possible

---

##  5. Native UI (Dioxus Desktop)

UI for configuration and visibility.

### Screens

* **Dashboard**

  * Current mappings
  * Cert status
  * Proxy status
* **Add/Edit Mapping**

  * Form with domain + port
  * SSL toggle
* **Logs**

  * Real-time proxy logs
  * Errors & warnings
* **Settings**

  * Default domain suffix (`dev.local`)
  * Use mkcert or internal certs
  * Automatic port redirect toggle

### UI Principles

* Clear state: Running / Stopped
* No YAML
* Instant feedback

---

##  6. Configuration Files (Human & Machine)

DevBind stores settings so project setups can be repeated.

### Format

TOML (`~/.config/devbind/config.toml`)

### Example

```toml
[proxy]
listen_port = 8443
use_mkcert = true

[[routes]]
domain = "dev.local"
port = 80

[[routes]]
domain = "sub1.dev.local"
port = 8085
```

---

##  7. CLI Companion

Optional lightweight CLI if you don’t want UI.

```bash
devbind start
devbind add sub8.dev.local 9090
devbind list
```

Same config as UI.

---

##  8. Logging & Diagnostics

Developers need to see what’s going on.

### Logs

* Proxy access logs
* Error logs (SSL, permission failures)
* Real-time UI viewer

#  **Example Use Cases**

## Case 1 — Simple Local HTTPS

1. Open DevBind
2. Add: `dev.local → 80`
3. Click “Enable HTTPS”
4. SSL generated by mkcert
5. Visit: `https://dev.local`

No nginx, no messy config.

---

## Case 2 — Multiple Services

You have:

```
sub1.dev.local → :8085
sub2.dev.local → :3210
sub3.dev.local → :3333
```

With one wildcard cert & proxy, everything works under HTTPS.
